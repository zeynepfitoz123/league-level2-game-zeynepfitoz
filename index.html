Fly Catchers

by: Zeynep Fitoz

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because it is a great distraction. It is engaging yet easy for anyone to play. The game is set in a swamp because I was very interested in the dynamics of the animals in a swamp.

2. What I Learned

Some feedback that I got was to make the game more fast-pace. I changed the code to make the flies fly faster over the frog. I also got feedback to make the left and right movability more useful.

3. Architecture of My Game

I have 6 classes and they all serve a very important purpose in my game. The FrogGame class compiles everything together and sets the size parameters for the game. The FrogGamePanel handles all things visual including the text, background, different panels, some aspects of the characters, and the keys that dictate the movements in the game. The FrogObjectManager controls the way objects and characters interact with each other and how that affects the game. The FrogGameObject class regulates the bounds of objects and their collision boxes. The FrogCharacter and FlyCharacter class create and control their corresponding character and the different colors they are.

4. Technical Implementation

I am very proud of my whole game but I am particularly proud of the different colors of my characters. When my frog's tongue catches a fly of the same color, the frog changes color and the player gains a point. This was very complicated because the flies had to continuously fly out in different colors and the frog character had to change the color it needed to gain a point each time the frog caught a new fly. To accomplish this I took advantage of ArrayLists and the different colors it could hold.

5. Most Challenging Part

The most challenging part of this project was figuring out the logistics of the frog's tongue. Although it seems simple, I had many different things to think about. This included how high the tongue should go, how it would go up, which flies would it accept, and how the tongue would match up with the other objects. The solution was a lot of trial and error.

6. Future Fixes and Updates

If I could improve my game, I would make the game harder. I would increase the speed of the flies and create objects that the frog will have to dodge.
